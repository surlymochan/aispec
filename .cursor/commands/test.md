# /test — 测试报告

**指令**：对照验收标准执行测试并产出 TEST 报告，置于当前迭代目录。

**输入**：实现与验收标准（PRD/迭代 README）。  
**输出**：当前迭代目录下的 `TEST.md`。

**执行步骤**：
1. 阅读当前迭代的 PRD 验收标准与迭代 README（若有验收脚本则执行）。
2. 执行测试或验收脚本，记录通过/失败项与结论。
3. 将测试结果写入当前迭代的 `TEST.md`。
4. 验收通过后可进入 /merge；若迭代约定验收脚本，/merge 前须通过。

**权威**：aispec spec/CONVENTIONS.md 第八节（8.1 表格 /test、8.4 验收）。

# /coding — 实现

**指令**：按 DESIGN 与 tasks.md 进行代码与配置等实现。

**输入**：DESIGN.md、可选 tasks.md。  
**输出**：代码与配置等实现产物，置于当前迭代目录或 src/（按项目约定）。

**执行步骤**：
1. 阅读当前迭代的 DESIGN.md 与 tasks.md（若有）。
2. 在迭代目录或项目约定的 src/ 下实现，不得污染项目主文件（src/spec/根目录）直至 /merge。
3. 所有变更控制在当前 SP 目录内，或按项目约定的「迭代内开发」路径放置。
4. 实现完成后可触发 /review、/test。

**权威**：aispec spec/CONVENTIONS.md 第二、八节（8.1 /coding；SP 与主文件隔离）。

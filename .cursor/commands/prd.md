# /prd — 产品需求文档

**指令**：基于 MRD 与澄清结果产出当前迭代的 PRD.md。

**输入**：MRD + 与负责人澄清结果。  
**输出**：`iteration/SP0xxx/PRD.md`（产品需求与验收）。

**执行步骤**：
1. 阅读当前迭代的 MRD.md；信息不足时先列出待确认项与负责人澄清，不得凭假设编写。
2. 编写 PRD：产品目标、用户与场景、功能需求（含验收）、非功能需求、验收标准摘要。
3. 将 PRD 写入当前迭代目录的 `PRD.md`。
4. 后续可触发 /design。

**权威**：aispec spec/CONVENTIONS.md 第一、八节（8.1 表格 /prd）；PRD、DESIGN 须先多询问、澄清再落笔。

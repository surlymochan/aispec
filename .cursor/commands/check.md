# /check — 发布检查清单

**指令**：执行发布前检查（脱敏、同步、在发布目录内测试、本地 commit）；常由 /public 脚本自动执行，也可单独触发核对。

**输入**：人触发或由 /public 脚本调用。  
**输出**：脱敏通过、同步完成、测试通过、已 commit（不 push）；未通过则中止并提示。

**执行步骤**：
1. 脱敏检查：拟发布内容无本机路径、硬编码密钥等。
2. 同步到发布目录（若未同步）。
3. 在发布目录内运行约定测试；未通过则不 commit、不 push。
4. 测试通过后可在发布目录内 git add & commit；/push 由人工执行。

**权威**：aispec spec/CONVENTIONS.md 第一节（发布检查清单 /check）；/public 与 /push 分离。

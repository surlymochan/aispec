# /deploy — 部署与验证

**指令**：使用 src 或迭代产物与部署脚本执行部署，记录结果与验证结论。

**输入**：src 或迭代产物、部署脚本。  
**输出**：部署结果与验证结论，可记录于迭代 README 或单独报告。

**执行步骤**：
1. 按项目约定选择「全部部署」（从 src+spec）或「单迭代部署」（该迭代目录内脚本，优先读取已合并的 src）。
2. 执行部署脚本，记录成功/失败与关键输出。
3. 将部署与验证结论写入迭代 README 或迭代目录下的单独报告。
4. 部署验证通过后可作为 /merge 的前置依据。

**权威**：aispec spec/CONVENTIONS.md 第四、八节（8.1 表格 /deploy）。

# 贡献指南

感谢对 aispec 的关注。以下为贡献流程与约定。

## 如何贡献

- **小改动**（错别字、文档澄清、脚本修复）：可直接提 PR，在描述中简要说明变更目的。
- **功能或约定变更**：建议先在本仓库以 **迭代（SP）** 方式讨论与落盘：
  - 在仓库 Issues 中提出想法或 MRD 要点；
  - 维护者或贡献者可在 `iteration/` 下开 SP（如 SP0004），按 CONVENTIONS 执行 /prd → /design → /coding → /review → /test，验收后合并到 `spec/` 或 `cli/`；
  - 再提 PR 将已验收的变更合并到主分支。
- **CLI 代码**：位于 `cli/`，遵循现有代码风格；新增命令请在 `cli/README.md` 或文档中说明。

## PR 约定

- 目标分支：`main`（或仓库约定的默认分支）。
- 标题与描述请说明**变更类型**（文档 / 约定 / CLI / 模板）与**简要原因**。
- 若涉及 CONVENTIONS 或工作流，请确认与现有约定无冲突，或已在迭代中达成共识。

## 行为准则

- 保持友好、就事论事；对不同工具与流程（如 OpenSpec）的对比请客观表述，不贬低其他项目。
- 争议或歧义以 **spec/CONVENTIONS.md** 为权威参考；若 CONVENTIONS 未覆盖，可提 Issue 或 PR 补充约定。

## 与迭代流程的关系

本仓库采用 **spec 驱动**：约定与模板的变更经 iteration 内 MRD/PRD/DESIGN 验收后再合并到 spec 或代码库。若你希望推动较大变更，欢迎先开 Issue 描述需求，维护者会协助对齐是否走 SP 流程。
